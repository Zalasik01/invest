# Regras do Projeto

- Todas as mensagens de commit devem ser escritas em português brasileiro (pt-br).
- Use o formato conventional commits em pt-br. Exemplos:
  - `feat: adicionar comando /investir`
  - `fix: corrigir parse de valor no comando`
  - `docs: atualizar README com instruções`
  - `refactor: extrair lógica de retry para módulo separado`
- Comentários no código devem ser em pt-br.
- Mensagens de log e console também em pt-br.
- Nomes de variáveis e funções em pt-br (camelCase).
