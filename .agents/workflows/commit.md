# Gerar Mensagem de Commit

// turbo-all

1. Sempre escreva mensagens de commit em **português brasileiro (pt-br)**
2. Use o formato **Conventional Commits**:
   - `feat:` para novas funcionalidades
   - `fix:` para correções de bugs
   - `docs:` para documentação
   - `style:` para formatação
   - `refactor:` para refatoração
   - `test:` para testes
   - `chore:` para tarefas gerais

3. Exemplos corretos:
   - `feat: adicionar comando /investir com botões inline`
   - `fix: corrigir parse de valor monetário`
   - `docs: atualizar README com instruções de instalação`
   - `refactor: extrair lógica de retry para módulo separado`
   - `chore: configurar template de commit em pt-br`

4. A descrição deve ser curta, direta e em letras minúsculas (exceto nomes próprios)
5. **NUNCA** escreva mensagens de commit em inglês
